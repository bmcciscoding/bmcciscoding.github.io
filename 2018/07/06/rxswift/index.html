# RxSwift

Rx 是一个系列，是通过 Observable<Element>形式的接口，实现泛型抽象的计算。它的目的是容易去组合一步操作和事件数据流。

KVO，异步操作，各种数据流都被抽象为序列（Sequence）。这就是为什么 Rx 简单，优雅，并且强力！😂

# Basic 基础概念

观察者模和序列是等价，这是理解 Rx 的关键。

每个一个 Observable 就只是一个序列，和 Swift 里的序列最大的优势是，可以异步接收元素。

1 - 1 - 1 - 1 - complete	// 序列完成
1 - 1 - 1 - 1 - error			// 序列报错
tap - tap - tap ...				// 可以把 Button 按下抽象为一个无限的序列

一旦序列报错，或者完成，将不能生成新的元素。

序列内部的资源，只有终止后才会被释放。如果想主动释放，需要调用 dispose。但是如果是无限序列，比如 Button tap，自己不会终止，需要开发者来决定调用 dispose 的时机，通常是加入 disposebag，并且使用 takeUntil 操作符。如果不太明白操作符 Operator，没关系，先记住，在 Rx 里，任何东西都需要抽象成序列来理解。

let seq = Observable.just(1)					// 序列
let seq = Observable.just([1, 2, 3])  // 序列

现在数据有了，我们如何拿到这些数据呢？就是通过订阅 subscribe。

let subscription = seq.subscribe { (event) in
    print(event)
}

总结一下：序列是数据的集合，序列可以订阅，序列直到终止才会释放资源。

# Disposing 丢弃

除了上面说到的 complete 和 error，当我们主动调用 dispose，也会释放所有资源。是不是马上释放，取决于序列在哪个 scheduler 订阅和在哪个 scheduler dispose。

序列会占用资源，直到终止后才会释放，但是有些时候比如进入某个 ViewController，创建了一个 N 个元素序列，然后我们推出这个 ViewController，这时我们需要提前释放，可是我们又不想手动调用 dispose。碰到这汇总情况，我们可以使用 DidsposeBag，把所有的订阅 subscription 都放到这个 bag 里，等到 bag dealloc 使，bag 会自动 dispose 所有在 bag 里的订阅，从而释放资源。

或者使用 Take Until 操作符。takeuntil 的只取某个数据前的数据。

let seq = Observable.just(1).takeUntil(self.rx.deallocated) // self is UIViewController

即这个序列在 ViewController dealloc 后终止。这时候说下操作符吧，可以理解为修饰符，可以对队序列做一些操作。这里可以查看所有都操作符 (opens new window)

# Subscription 订阅

一个序列，只有被被订阅后，才会发生产数据。也就是说，创建序列时，只是定义了数据如何发送。只有被订阅了，才会真正的生产出来。换句话说，每被订阅一次，都会新创建一次数据

let seq = Observable<Double>.create { (ob) -> Disposable in
 		print("subscribe")
		ob.onNext(Date.init().timeIntervalSince1970)
  	ob.onCompleted()
	  return Disposables.create {
      	print("dispose")
	  }
}

seq.subscribe { print($0)}  // next(1593087313.581224)
seq.subscribe { print($0)}  // next(1593087313.581433)
seq.subscribe { print($0)}  // next(1593087313.581468)

每次订阅，创建的数据都是全新的。但是有时候，我们并不希望这样，比如数据是一个网络请求，如果我们订阅了多次，却需要发起多次网络请求，这是不太合理的。
