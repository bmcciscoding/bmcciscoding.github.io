iOS 关于布局的问题

Wed Jan 15 2020
  • 布局

这是我关于 iOS 布局相关的所有吐槽！但也是我局限的理解，欢迎各路英豪斧正！

# 现有的布局方式

官方支持的布局方式有 frame、autolayout、stackview。

# Frame

frame 布局宛如裹脚布，又臭又长。唯一的优点就是执行效率高，但是编码效率低，写起来很难受。而且容易造成很多魔数，让人维护起来很抓狂。在布局会动态变化，safe area 等一些情况，需要考虑很多方面。 在编码方面，有些方案会通过 category，给 UIView 添加一些常用的属性来避免直接设置 frame，也算是一个小小的改进书写体验。

@property (nonatomic) CGFloat left;        ///< Shortcut for frame.origin.x.
@property (nonatomic) CGFloat top;         ///< Shortcut for frame.origin.y
@property (nonatomic) CGFloat right;       ///< Shortcut for frame.origin.x + frame.size.width
@property (nonatomic) CGFloat bottom;      ///< Shortcut for frame.origin.y + frame.size.height
@property (nonatomic) CGFloat width;       ///< Shortcut for frame.size.width.
@property (nonatomic) CGFloat height;      ///< Shortcut for frame.size.height.
@property (nonatomic) CGFloat centerX;     ///< Shortcut for center.x
@property (nonatomic) CGFloat centerY;     ///< Shortcut for center.y
@property (nonatomic) CGPoint origin;      ///< Shortcut for frame.origin.
@property (nonatomic) CGSize  size;        ///< Shortcut for frame.size.

但是无法自适应布局，仍然让我很绝望。

推荐场景：固定的局部布局

# Autolayout

autolayout 是官方提供的，基于约束计算的布局方式。计算方式是基于 实现的。从 iOS12 之后，调整了约束计算的策略，提高了效率。但是依然不够美好。

v.addConstraint(NSLayoutConstraint.init(item: v,
                                        attribute: .top,
                                        relatedBy: .equal,
                                        toItem: self.view,
                                        attribute: .top,
                                        multiplier: 1,
                                        constant: 0))

如果你直接徒手撸，就要使用这样的 API，难受呀。业界也推出了 Masonry (OC) 和 SnapKit (Swift) 这两库，从 API 让开发者愉悦。但是虽然自动布局能解决大部分问题，但是有些地方使用起来还是相当繁琐。比如连续N个间距不同的视图，写起来还是很恶心。

# StackView

StackView 在我看来是，是个残次品，只能解决某些问题，而且每个版本支持的 api 都不一样，所以使用起来还是还让我苦恼。 比如这个接口，只支持 iOS11。

- (void)setCustomSpacing:(CGFloat)spacing afterView:(UIView *)arrangedSubview API_AVAILABLE(ios(11.0),tvos(11.0));

# YogaKit

除了以上官方提供的三种布局方式，业界还有 facebook 推出的 yoga 框架。使用了前端很熟悉的 flexbox 来进行布局。
